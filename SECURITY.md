# Reporting Security Issues

If you discover a security vulnerability:

1. Do not publish it publicly.
2. Report it to the project maintainer.
3. Include detailed reproduction steps.
4. Allow reasonable time for fixing the issue.

---

# Secure Development Rules

- Never upload passwords.
- Never upload API keys.
- Never upload private tokens.
- Never upload confidential data.
- Never commit personal information.
- Never commit secret configuration files.

---

# Repository Protection

Every repository should use:

- Branch Protection
- Pull Request Review
- Commit History
- Version Control
- Documentation
- License
